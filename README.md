# Orbital

Live demo: https://craigpaskowitz.github.io/Orbital/

A real-time space tracking interface exploring how live data can be turned into intuitive, engaging, and mobile-friendly experiences.

---

## Why this exists

Most real-time datasets are technically rich but hard to interpret.  
Orbital is an exploration of how to make live data—specifically near-Earth objects—feel more visual, interactive, and accessible.

This started as a simple question:
> What does a lightweight, real-time interface feel like when it’s designed for clarity instead of complexity?

---

## What it does

- Tracks near-Earth objects using live data  
- Visualizes position and movement in real time  
- Presents information in a clean, minimal interface  
- Optimized for mobile browsing (especially iPhone)

## How it works

- Single-page application (vanilla HTML, CSS, JavaScript)  
- Runs entirely client-side  
- Pulls from publicly available space data APIs  
- Deployed via GitHub Pages  

No backend, no frameworks—intentionally lightweight.

---

## Design approach

- Prioritize **immediacy** (data updates feel live)
- Keep the interface **minimal and readable**
- Design for **mobile-first interaction**
- Focus on **visual understanding**, not raw data density

---

## Notes & observations

- Real-time data creates a fundamentally different user experience than static datasets  
- Mobile constraints force better prioritization of information  
- Even simple visual layers significantly improve comprehension  
- Lightweight, client-side builds are often sufficient for surprisingly rich experiences  

---

## Future ideas

- More contextual data layers (size, distance, risk indicators)
- Time-based playback / rewind
- Enhanced visual cues for movement and proximity
- Additional real-time datasets beyond space objects
